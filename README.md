# Version Control and Repository Management using Git and GitHub

## 📌 Description

This project demonstrates the basic steps involved in using Git and GitHub for version control and repository management. It includes creating an account, setting up a repository, and uploading files.

## 🎯 Aim

To understand the fundamentals of version control using Git and GitHub, including creating repositories and managing files efficiently.

## 🛠️ Tools Used

* Git
* GitHub
* Web Browser

## 📂 Steps Included

### Step 1: Create a GitHub Account

1. Go to GitHub website.
2. Click on Sign Up.
3. Enter username, email, and password.
4. Verify your email and create the account.

### Step 2: Create a New Repository

1. Click on "+" icon in the top-right corner.
2. Select "New repository".
3. Enter repository name.
4. Choose Public or Private.
5. Initialize with README.
6. Click "Create repository".

### Step 3: Upload a File

1. Open the repository.
2. Click "Add file" → "Upload files".
3. Choose files to upload.
4. Add commit message.
5. Click "Commit changes".

## 🗑️ Deleting Files

Files (including README.md) can be deleted either:

* Using GitHub interface (trash icon)
* Using Git commands (`git rm filename`)

## ✅ Conclusion

This project helps in understanding how Git and GitHub are used for tracking changes, managing code, and collaborating on projects.
